Source Code Management System

30 Jan 2014 1 Comment

Dalam pengembangan aplikasi, maka source code merupakan basic building blocks. Aplikasi tidak dibuat dalam satu kali pekerjaan, namun dalam beberapa tahap. Sama seperti kita membangun sebuah rumah, maka kita mulai dari fondasi, kemudian tembok dan terakhir pintu dan jendela. Oleh karena itu, penting bagi bagi kami agar source code dapat kami telusuri perkembangan-nya dari awal hingga akhir.

Selain itu, karena ada beberapa developer/programmer atau designer yang bekerja pada satu aplikasi, maka perubahan yang dilakukan oleh satu developer/designer seharusnya tidak terhapus oleh perubahan oleh developer/designer lainnya.

Disinilah, source code management system diperlukan. Saat ini sudah banyak source code management system yang tersedia. Tiga yang paling populer adalah : CVS, Subversion atau disingkat SVN dan GIT.

GIT merupakan source code management system yang terbaru dan melakukan pendekatan yang berbeda dengan pola “distributed” dimana tidak ada sebuah “centralized server” seperti yang digunakan di CVS dan SVN.

CVS sendiri saat ini sudah kurang populer dan jarang digunakan. SVN merupakan “penerus” untuk CVS karena merupakan pengembangan dari sistem CVS, khususnya dalam hal folder management.

Idea Lounge sendiri menggunakan SVN,  menggantikan sistem CVS yang sudah digunakan sejak tahun 2000.

Pola kerja SVN sendiri dimulai dari:

A. Pembuatan Repository.

Di tahap ini, sebuah lokasi dibuat di server dan diberi nama yang unik dimana nama tersebut akan digunakan sebagai penanda akses oleh developer/designer. Umumnya nama yang digunakan adalah nama aplikasi yang akan dibuat.

B. Proses Checkout.

Pada proses ini, maka developer akan men-sinkronisasi-kan data dengan download dari server dan membuat sebuah “local copy” di komputer developer/designer.

C. Proses Update.

Nah, proses update ini dilakukan untuk memperbaharui source code di local copy dengan source code baru di server yang sudah dicommit oleh developer/designer lain. Hal ini penting untuk dilakukan sebelum proses commit dilakukan agar tidak terjadi konflik dimana source code di server dan source code di local copy berbeda.

Sistem SVN sudah dapat melakukan merger source code secara otomatis dimana perubahan otomatis dilakukan oleh sistem di proses update ini. Dengan catatan, perubahan terjadi hanya di penambahan atau pengurangan baris source code.

Bila dalam satu baris yang sama terdapat pertentangan source code, maka developer/designer wajib melihat dan memperbaiki agar source code menjadi selaras. Tentu saja hal ini dilakukan dengan tetap menjaga source code hasil commit developer/designer lain tetap dapat berjalan dengan baik. Demikian juga dengan perubahan yang dilakukan oleh developer/designer yang melakukan update.

D. Proses Commit.

Nah, setelah melakukan checkout, maka developer/designer akan melakukan coding.  Apakah itu untuk menambahkan fitur baru atau membetulkan bug. Setelah selesai, maka dilakukan proses commit dimana sistem akan menambahkan source code baru ke dalam repository.

E. Perulangan

Proses perulangan yang terjadi adalah di tahap C dan D. Dimana tahap A dan B hanya perlu dilakukan di 1x di awal.

Namun untuk developer/designer baru tetap perlu melakukan tahap B agar mendapatkan local copy.

Kelebihan dari SVN adalah:

  1. Memungkinkan kolaborasi antar developer. Dimana kode yang sudah di-commit ke dalam repository akan tetap dijaga oleh developer berikutnya.
  2. Adanya riwayat source code. Setiap kode yang di-commit akan memiliki satu penomoran sendiri, dimana akan dicatat oleh sistem file/folder apa yang berubah dan siapa yang melakukan perubahan.
  3. Akses ke repository terbatas. Dimana untuk mengakses repository diperlukan username dan password. Otomatis pengguna SVN wajib didaftarkan terlebih dahulu. Sehingga tidak sembarang orang bisa mengakses dan melakukan perubahan.

Nah, untuk SVN Client sendiri ada banyak aplikasi. Namun yang populer digunakan adalah:

  1. RapidSVN untuk Linux
  2. TortoiseSVN untuk Windows

Masih banyak lagi aplikasi SVN Client yang bisa digunakan, silahkan gunakan mesin pencari Google untuk mencarinya. Atau mungkin tertarik untuk menelusuri lebih lanjut ?
